# 落札実績オープンデータ活用 設計メモ(フェーズ2)

**ステータス**: 設計のみ(実装未着手)
**データソース実測日**: 2026-07-11(取得・列構成確認済み)

## データソース仕様(実測)

- **取得元**: 調達ポータル公式オープンデータ(デジタル庁)
  - 一覧ページ: `https://www.p-portal.go.jp/pps-web-biz/UAB02/OAB0201`
  - 実ファイルURL: `https://api.p-portal.go.jp/pps-web-biz/UAB03/OAB0301?fileversion=v001&filename=<ファイル名>`
- **ファイル体系**:
  - 年度別全件: `successful_bid_record_info_all_<年度>.zip`(2013年度〜、0.1〜1.8MB)
  - 日次差分: `successful_bid_record_info_diff_<YYYYMMDD>.zip`(直近2ヶ月分のみ保持、実測94行/日程度)
- **CSV形式**: UTF-8(BOM付き)、ヘッダー行なし。列(実測順):
  `案件ID, 案件名, 落札日, 落札金額, 種別コード, 機関コード, 落札者名, 法人番号`
- **認証・申請**: 不要。**商用利用可・出典表記必須**(調達ポータルサイトポリシー)
- **注意**: オープンカウンタ(少額)案件の落札結果を含むが、個人事業主の落札分は
  含まれない(公式注記)。GEPS経由案件が対象であり、地方自治体の独自入札は対象外

## 要調査(実装前に潰す)

- 種別コード(S1/S5/P1/L1等)・機関コードのコード表の所在。公式のコード定義が
  見つからない場合は、案件名・落札者から推定せず「コード生値のまま保持+表示は
  機関コード→機関名の手引きが得られた範囲のみ」とする
- 全件ファイルのヘッダー有無・列順が差分と同一かの確認(年度またぎの形式変更に注意)

## アーキテクチャ方針: ステートレス取得(DB持たない)

年度別全件ZIPが最大1.8MBと軽量なため、**永続ストアを持たず実行時に必要な年度分を
ダウンロードして使う**。GitHub Actionsのステートレス実行と相性が良く、
インフラ追加ゼロ。応答性が必要になった段階で初めてキャッシュ(リポジトリ内
parquet等)を検討する。

新規モジュール `modules/awards.py`(フェーズ2):

```
fetch_awards(fiscal_years: list[int]) -> list[AwardRecord]
    年度別全件ZIPを取得しAwardRecord(Pydantic)にパース

find_comparables(keywords: list[str], records) -> list[AwardRecord]
    案件名のキーワード部分一致で同種案件を抽出

price_stats(comparables) -> PriceStats
    件数 / 中央値 / 四分位(p25-p75) / 直近の実例3件(案件名・金額・落札者)
```

## 商品への組み込み(3段階)

### ① レコメンドへの「参考落札相場」欄(premium向け・最初に作る)

日次バッチのマッチ結果に対し、キーワードで同種の過去落札を引き、
メール・シートに1行追加する:

> 参考落札相場: 同種12件 / 中央値 ¥248,000(p25 ¥180,000〜p75 ¥310,000)

- マッチ度スコアには影響させない(情報提供のみ。相場が薄い品目で0件でも減点しない)
- 同種判定は案件名キーワードの部分一致から始める(過剰な曖昧マッチはさせず、
  0件なら「相場データなし」と正直に表示する)

### ② 見積ドラフトの価格根拠(フェーズ2本体)

既存mitsumori-createスキルの選定基準「過去の同種案件の落札価格があればその範囲内で
バランス案を調整する」を、このデータで機械的に裏付ける。安全案/バランス案/攻め案の
算出時に、同種案件の落札金額分布を「価格レンジの妥当性チェック」として併記する。
**原価割れ禁止・最低粗利5%の絶対ルールは変更しない**(相場が原価×1.053を下回る
場合は「応札非推奨」フラグを出す)。

### ③ 週次サマリの「落札相場ウォッチ」(差分データ活用)

日次差分CSV(軽量)を週次で束ね、顧客の対象キーワードに関する新規落札を
「今週の同業界の落札動向」として週次レポートに掲載する。

## 競争上の位置づけ

NJSSの下位プランでは落札相場の分析提供が手薄であり、本機能は
「NJSSより安いのに相場が見える」という差別化の本命。データは公式・無料・
規約クリーンで、資格情報リスクもゼロ。

## 実装上の注意

- **出典表記**: 顧客向けメール・シート・レポートに「出典: 調達ポータル
  (デジタル庁)落札実績オープンデータ」を必ず記載する(利用条件)
- 落札金額は税込/税抜の別が明示されていない可能性があるため、表示時は
  「公表値」として扱い、当社で税計算の加工をしない
- ダウンロードは1実行あたり数ファイルに留める(全年度一括取得を毎日はしない。
  相場計算は直近2〜3年度分で十分)
- 工数目安: ①が小(1〜2日)、②はフェーズ2本体と同時、③は小
