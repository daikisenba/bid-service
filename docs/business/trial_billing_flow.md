# トライアル&決済導線 設計案(Stripe Payment Link・ノーコード運用)

**ステータス**: 提案(実装未着手)。Payment Link URL の発行後にLPへ組み込む。
**方針**: サイトにStripe API・DB・サブスク状態管理は組み込まない(引き算の思想)。
決済はStripe Payment Linkに全委任し、配信の有効化は既存の顧客マスタ(ステータス列)で管理する。

---

## 0. 前提の推奨決定

| 項目 | 推奨 | 理由 |
|---|---|---|
| トライアル期間 | **14日間** | 配信は平日朝のみ=7日だと実質5通(マッチ0の日もあり得る)。14日なら約10通で価値を体感させられる |
| 課金 | 月額33,000円(税込)・カード必須トライアル | Stripe Payment Linkの標準機能(無料トライアル日数設定)で実現。コード不要 |
| 決済手段 | クレジットカードのみ | 方針どおり。請求書払いを求める企業は個別相談に回す(無理に対応しない) |

### Stripe側で行う設定(あなたの作業・15分)

1. Stripeダッシュボード → 商品を作成: 「入札秘書 月額プラン」33,000円(税込)/月・継続
2. Payment Link を作成: 上記価格 + **「無料トライアル 14日」を有効化**(カード情報必須)
3. 設定で推奨: ①メール領収書ON ②**カスタマーポータルON**(顧客が自分で解約できる=解約対応の手間と心理的安心の両方に効く) ③明細表記(statement descriptor)を「SOUKI」等に
4. 発行された Payment Link URL を私に共有 → LPに組み込みます

### ⚠️ 公開前に必須の法令対応(ここだけは省略不可)

- **特定商取引法に基づく表記ページ**: クレカでオンライン販売する以上、法的に必須。
  事業者名/所在地/連絡先/料金/支払方法/解約条件を記載した1ページをサイトに追加する
  (実装は10分。文言はテンプレを用意できます)
- **自動課金の明示**: 「トライアル終了後は自動的に月額33,000円(税込)の課金が開始」を
  ボタンの直近に明記(2022年特商法改正の定期購入規制対応。Stripe Checkout側でも
  表示されるが、当社ページでも明示するのが誠実でクレーム予防になる)

## ① 画面導線(UX)設計 — CVR最大化の構成

### 基本思想

- **「無料トライアル」を選んだ人には、成功画面で即カード登録に進ませる**(熱が最も高い瞬間を逃さない。後日メールでURLを送る方式は開封されず数割落ちる)
- **「資料請求」を選んだ人には資料DLを主役に、トライアルは控えめな次アクション**として添える(温度感に合わない売り込みはCVRを下げる)
- 不安の除去を3点セットで並記: 「期間中の解約は0円」「いつでも解約可」「登録は約1分」

### 成功画面A: 「無料トライアル」を選んで送信した場合

```
✅(緑チェック)
送信ありがとうございました！

このままクレジットカードをご登録いただくと、
14日間の無料トライアルを今すぐ開始できます。

┌──────────────────────────────────┐
│ ▶ 14日間無料トライアルを始める        │ ←Stripe Payment Link(青・最大)
│   （カード登録・約1分）              │
└──────────────────────────────────┘
・トライアル期間中の解約は料金0円
・トライアル終了後は月額33,000円(税込)が自動開始されます
・解約はいつでも可能です(お客様ご自身で完結)

───(区切り)───
サービス資料をPDFで見る(控えめなテキストリンク)
「まずは相談したい」方はそのままお待ちください。
代表の千羽(d.senba@souki-cp.co.jp)よりご連絡します。
```

### 成功画面B: 「資料請求」を選んで送信した場合(現行を拡張)

```
✅ 資料請求ありがとうございました！
[サービス資料をダウンロードする](現行の青ボタン・主役のまま)

───(区切り)───
📩 実際のレコメンドメールを体験したい方へ
[14日間無料トライアルを始める(カード登録・約1分)]
 ・期間中の解約は0円 / 終了後は月額33,000円(税込)
```

### 補足(CVRの正直な見立て)

- カード必須トライアルは登録率が下がる代わりに**有料転換率が高い**(数字が読める)。
  最初の10社はこの方式で「転換率」を測るのが正しい。登録率が想定より低ければ、
  そのとき初めて「カード不要トライアル(手動請求)」を検討すればよい
- LPヒーローの「無料トライアルを試す」ボタンも、将来的にPayment Link直リンクに
  変えられるが、**まずはフォーム経由に統一**を推奨(条件ヒアリングの機会を失わない)

## ② トライアル開始後の「配信有効化」運用設計

### 全体フロー(半自動・1顧客あたり作業10分+毎朝1分)

```
顧客: フォーム送信 ──→ Stripeでカード登録(トライアル開始)
                              │
            Stripe → d.senba@ に「新規サブスクリプション」通知メール(自動)
                              │
あなた: (1) 顧客へ条件ヒアリング(フォームの記載で足りなければメール1往復)
        (2) setup_customer_sheet.py で顧客登録 + ステータス=active
                              │
翌平日朝: bid-service が自動でレコメンド生成 → あなた宛に確認メール
        → 内容確認して顧客へ転送(フェーズ1の設計どおり・誤配信ゼロ運用)
```

### 突合の仕組み(DBなしでズレを防ぐ)

- **照合キーはメールアドレス**(フォームとStripe登録で同じアドレスを使ってもらう。
  成功画面に「※カード登録は、フォームと同じメールアドレスでお願いします」と一言添える)
- 顧客マスタの既存列をそのまま流用:
  - `契約開始日` = トライアル開始日(Stripe通知の日付)
  - `次回請求日` = トライアル終了日(Stripeが計算した初回課金日をそのまま転記)
  - `プラン` = standard固定(一律プランなので分岐なし)
- **週1回5分の突合**: Stripeダッシュボードのサブスク一覧と顧客マスタのactive行を
  見比べる。「Stripeにあるのにシートにない」(登録漏れ)と「シートでactiveなのに
  Stripeで解約済み」(止め漏れ)の2つだけチェック

### 解約・支払い失敗時

- Stripeから通知メールが来る → 顧客マスタのステータスを `paused` に変更(1分)。
  翌朝から配信対象外になる(既存のフィルタがそのまま効く)
- カスタマーポータルONにしておけば、解約はお客様自身で完結し、あなたは
  ステータス変更だけすればよい

### この運用の損益分岐(正直な見積もり)

- 1顧客あたり: 初期登録10分 + 毎朝の転送確認1分 + 週次突合5分/全体
- **10社まではこの手動運用が最速・最安**。自動化(Stripe Webhook→GAS等)は
  10社を超えて毎朝の転送が苦になった時点で初めて作る(いつもの原則どおり、
  需要が証明されてから投資)

### 実装が必要になる箇所(私の作業・Payment Link URL受領後)

1. LP成功画面A/Bの実装(上記の構成どおり。半日以内)
2. 特商法表記ページの新設(テンプレ文言込みで1時間)
3. (任意・小)顧客マスタ処理の対象ステータスに関する整理 — 現行コードは
   `active` のみ配信対象。トライアル中も `active` にする運用で問題ないが、
   「トライアル中」を見分けたければ `次回請求日` 列で判別する(コード変更不要)
